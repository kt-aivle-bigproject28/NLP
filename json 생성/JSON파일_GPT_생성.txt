조건은 다음과 같습니다:

text_data에는 업로드한 텍스트 파일의 내용이 수정 없이 전부 포함되어야 합니다.
cuts 부분에는 제공된 시나리오 내용을 정확히 반영해야 합니다.
scene은 포함하지 않으며, cuts와 text_data를 분리하여 작성해야 합니다.
생성되는 JSON 파일의 이름은 업로드한 텍스트 파일 이름과 동일해야 합니다.
다운로드 가능한 링크를 반드시 제공해야 하며, 클릭 시 파일이 정상적으로 다운로드되어야 합니다.

JSON 형식 유지

text_data: 전체 텍스트 내용.
cuts: 배열로, 각 장면(cut)에 대한 정보 포함.
cuts 배열 내부 구성

cut_id: 장면의 고유 ID (1부터 시작).
location: 장면의 장소를 간결하게 작성.
caption: 장면의 설명 또는 주요 행동.
dialogue (선택적): 해당 장면에서의 대화. 필요할 경우 포함.
텍스트 기반

text_data는 원본 텍스트 데이터를 포함.
각 cut는 특정 위치 및 관련된 설명을 포함.
일관성 유지

입력받은 형식과 동일하게 출력해야 함.
cuts 배열에 포함된 정보는 입력 내용에 기반하여 변형 없이 생성.
